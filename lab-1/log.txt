Q: why can you use only new() to make a list

A: ecause C# can infer the type from the left side.
When you write:
public static List<Pickup> coinPickups = new() { ... };
the compiler already knows coinPickups is List<Pickup>, so new() is treated as:
new List<Pickup>()
This is called target-typed new.
You can use new() only when the target type is clear, for example:
•	variable declaration with explicit type
•	field/property with explicit type
•	method argument where parameter type is known
You cannot use new() if the compiler cannot figure out the type.
Example:
var x = new(); ❌ (no target type)
